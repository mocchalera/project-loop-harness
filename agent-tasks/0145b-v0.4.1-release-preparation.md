# 0145b: v0.4.1 local release preparation

- **Status:** Approved release slice
- **Milestone:** v0.4.1 Integrity Migration
- **Priority:** P0
- **Estimated size:** M
- **Dependencies:** 0141-0145a complete
- **DB schema:** remains 8

## Goal

Prepare a locally verified v0.4.1 release candidate after the complete
advisory-to-enforced migration dogfood. Stop before every externally visible
publication action.

## Scope

- Bump package/runtime version metadata from 0.4.0 to 0.4.1.
- Add `docs/release-notes/v0.4.1.md` covering 0141-0145a, compatibility,
  dogfood evidence, known advisory findings, and publication status.
- Ensure `TASKS.md`, the agent-task index, the release plan, and security
  support line are internally consistent.
- Run the complete local release checklist: Ruff, full tests, scratch-project
  strict validation/render, sdist/wheel build, Twine checks, extracted-sdist
  contract, and clean fresh-wheel smoke.

## Invariants

- No runtime behavior change is part of release preparation.
- Schema remains 8; dependencies, completion-packet/v1,
  handoff-packet/v1, and fresh-project lifecycle defaults are unchanged.
- Existing-project lifecycle policy remains advisory unless its operator
  explicitly enables enforcement.
- Do not modify repository `.project-loop`, `.claude`, `.work`, or unrelated
  user files.
- Do not push, tag, create a GitHub Release, upload to PyPI, update pipx, or
  perform any other externally visible publication step.

## Acceptance criteria

- `pcl --version` from source reports 0.4.1 and old-version grep finds no stale
  package/test metadata outside historical documentation.
- Release notes name every v0.4.1 task and honestly describe the completed
  v0.3.0-to-v0.4.1 migration dogfood.
- Ruff and the full suite pass from the canonical source tree.
- A scratch project validates strictly at schema 8 and renders.
- Fresh sdist and wheel pass Twine, extracted-sdist contract coverage, and a
  clean-venv wheel install/init/strict-validate/render smoke.
- An independent read-only reviewer approves the release diff and evidence.

## Evidence required to close

- Local release commit and exact changed files.
- Test counts and package artifact names/hashes.
- Scratch, extracted-sdist, and wheel-smoke outputs.
- Explicit statement that external publication remains unperformed and
  requires human authorization.
