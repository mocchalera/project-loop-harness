# project-loop-harness v0.5.3

**Prepared:** 2026-07-20

**Status:** Published and publicly verified

**Scope:** strict Evidence integrity and target-bound harness feedback

This release candidate makes copied Evidence and Evidence Set resolution more
defensive, and adds a bounded Gap Report feedback contract for diagnosing the
earliest failed handoff. Integrity checks prove artifact identity and binding;
they do not turn producer claims into verified facts.

## Highlights

### Event-anchored strict Evidence resolution

- Copied adhoc Evidence resolves against its immutable recording event,
  manifest metadata, canonical stored path, exact size, and SHA-256 before
  returning bytes.
- Evidence Sets receive the same event-anchor, canonical-path, regular-file,
  identity, size, and exact-byte validation boundary.
- Missing, ambiguous, redirected, substituted, or drifted artifacts fail
  closed with structured findings instead of silently trusting mutable paths.
- These resolvers establish integrity only. Relevance, sufficiency, reviewer
  independence, and factual correctness remain separate verdict concerns.

### `gap-report/v1` feedback Evidence

- `pcl gap add/show/list/promote` records the earliest failed handoff, one
  closed gap classification, and target-bound candidate lessons as immutable
  Evidence.
- Reports may target Goals, Tasks, Features, Defects, Workflow Runs, and Agent
  Jobs. Related Evidence and Workflow Run references are validated before
  mutation.
- Candidate-lesson promotion requires human-origin provenance and supporting
  Evidence, remains hash-bound and idempotent, and records
  `application_status: pending`; PLH never edits a durable owner automatically.
- Reports remain producer-authored claims. `worker_limitation` is explicitly
  provisional and one trajectory cannot establish a general limitation.

### Gap Report integrity hardening

- Artifact creation uses canonical no-follow directory handles, exclusive
  temporary files, and non-overwriting publication so directory, temp, and
  final symlinks cannot redirect writes outside the project.
- `artifact_sha256` binds the exact stored UTF-8 bytes, detecting same-size
  whitespace-only changes as well as semantic tampering.
- Schema and runtime validation agree on real UTC calendar dates. Candidate
  lessons are keyed by structurally unique lesson IDs, and duplicate raw JSON
  keys are rejected while loading.
- Gap-class filters use the immutable class recorded in the anchor event and
  expose both recorded and current artifact values so tampering cannot hide a
  warning in another filter bucket.

## Compatibility and boundaries

- DB schema remains **8**; no migration is added.
- Runtime dependencies remain empty and Python support remains 3.10-3.13.
- `gap-report/v1` is new since v0.5.2; no public prior Gap Report artifact
  shape is being migrated.
- Core remains local-only. No telemetry, hosted state, provider execution,
  automatic durable-owner write, or external publication action is added.
- Publication does not change the local-only runtime boundary: no telemetry,
  hosted state, provider execution, or automatic external mutation was added.
