# v0.3.0 - Target-Bound Context

Project Loop Harness v0.3.0 turns the code-context receipt from a convenient
"latest receipt" lookup into a target-bound handoff contract: a context pack can
be bound to a specific task or job, backed by queryable link state rather than
an artifact scan.

Highlights:
- Generic `evidence_links` table (schema migration 007). Evidence can be linked
  to a target (`task`, `agent_job`) with a `link_role` (`supporting`,
  `code_context`). Existing `evidence.linked_task_id` rows are backfilled as
  role `supporting`; the column is retained for 0.x compatibility. Strict
  validation reports dangling links for known target types.
- Target-bound code context receipts. `pcl impact --diff --for-task T-XXXX` and
  `pcl impact --diff --for-job J-XXXX` record a caller-asserted binding both as
  a `code_context` evidence link and in the `context-receipt/v0` artifact
  (`target_binding`), written in one transaction so the row and the artifact
  agree.
- Target-bound context pack selection. `pcl context pack --include-code-context`
  now prefers the newest receipt bound to the pack's target, selected by a SQL
  query over `evidence_links`. `code_context.relevance` reports
  `scope: target_bound` / `binding_strength: caller_asserted` for a matching
  bound receipt, and keeps `unscoped_latest` / `missing_receipt` for the
  fallback states.
- `pcl context pack --require-bound-receipt` fails with a typed
  `context_pack_bound_receipt_required` error (with a target-specific refresh
  suggestion) instead of silently falling back to an unrelated latest receipt.
- Copied-evidence source-drift detection now also warns when a source file
  changes content without changing byte length (`source_drifted` with
  `detail: hash_mismatch`), closing the same-size gap left by v0.2.4.
- Six fixture-backed contract tests freeze the v0.3.0 code-context handoff
  contract so future changes stay provably additive.

Binding is a caller assertion, not a semantic-relevance proof. `pcl` records
that a receipt was created for a target, with the staleness facts at that time;
it does not claim the receipt is sufficient or that any agent read or understood
it. No cognition or go/no-go vocabulary is added.

Compatibility:
- Schema migration 007 (`evidence_links`), additive; schema version 6 -> 7.
  Never-downgrade and DB-ahead typed rejection are preserved; an older `pcl`
  binary typed-rejects a v7 database.
- `evidence.linked_task_id` is retained; existing task-linked evidence keeps
  working and is backfilled into `evidence_links`.
- No new runtime dependencies.

Shipped specs: `agent-tasks/0113` (evidence_links table), `agent-tasks/0108`
