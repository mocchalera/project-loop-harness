# project-loop-harness v0.4.2

**Prepared:** 2026-07-11
**Status:** Local release candidate; not published
**Scope:** Adaptive Entry

This release adds an optional, deterministic entry layer that can preserve a
reviewed Work Brief, recommend Direct/Discover/Assure, explain multi-axis
execution policy, and record an explicit audited override. The existing
`pcl start -> finish -> resume` path remains valid without adopting these
optional surfaces.

## Highlights

### Immutable Work Brief Evidence (0146)

- `work-brief/v1` captures intent, acceptance criteria, constraints,
  non-goals, and assumptions as immutable Evidence.
- `pcl brief add` validates and records the artifact; `pcl brief approve`
  records a separate actor/reason approval event bound to the artifact hash.
- Context and resume surfaces expose only approved metadata and Evidence
  references. Assumptions remain claims, not facts.

### Deterministic route recommendation (0147)

- `pcl route recommend` returns Direct, Discover, or Assure with normalized
  signals, stable reason codes, risk level, and an input digest.
- Recommendation is read-only by default. `--record` is an explicit,
  idempotent Evidence/event mutation.
- Auth/permission, migration, dependency, missing-acceptance, assumption, and
  deterministic-check signals are resolved without calling an LLM.
- Model self-assessment is never accepted as a risk-lowering signal.

### Explainable adaptive policy (0148)

- `pcl policy resolve` and `pcl policy explain` map a route onto planning,
  verification, chunk size, checkpoint, context, tool, wall-time, and
  escalation axes.
- Resolution precedence is defaults, profile preset, matching project rules,
  then non-overridable risk floor.
- Every axis reports its source. Conflicting rules fail closed instead of
  depending on list order.
- R2–R4 verification/checkpoint/chunk floors cannot be weakened by custom
  policy.

### Audited override and packet integration (0149)

- `pcl route override --dry-run` previews without mutation; apply requires an
  explicit target, profile, actor, and reason.
- Apply preserves the original recommendation and original policy resolution
  as separate hash-bound Evidence, then writes `route-override/v1`.
- The three Evidence rows/links and one aggregate event/outbox pair commit in
  one transaction. Exact repeats are idempotent.
- Permission, migration, destructive-operation, human-review, and R4 route
  floors reject downgrade attempts.
- `pcl route current` checks referenced artifact hashes before returning the
  historical original/effective view.
- Optional adaptive-route references flow through task context, completion
  packets, and resume handoffs without invalidating older fixtures.

## Compatibility and boundaries

- DB schema remains **8**; there is no migration in v0.4.2.
- Runtime dependencies remain empty.
- Work Brief, route, policy, and override adoption is optional.
- Existing `completion-packet/v1`, `handoff-packet/v1`, and `context-pack/v1`
  readers remain compatible; new adaptive-route fields/references are
  optional and additive.
- The packaged default policy is strict JSON and uses Python standard-library
  loading.
- No hosted backend, cloud sync, telemetry, paid service, automatic model
  selection, or automatic agent launch is added.
- Package, CLI, and MCP version metadata change from 0.4.1 to 0.4.2. The
  independently versioned Codex plugin manifest is unchanged.
- Task IDs shipped: 0146, 0147, 0148, 0149, 0149a, and 0149b.
