# project-loop-harness v0.5.4

**Prepared:** 2026-07-22

**Status:** Local release candidate; not published

**Scope:** behavior-preserving CLI and command-service maintainability

This patch release packages the completed post-v0.5.3 refactor that splits the
large CLI dispatcher, parser construction, and command services into bounded
modules while retaining the existing public and runtime contracts.

## Highlights

### Bounded command handlers

- Goal, Task, Feature, Story, Test, and Defect orchestration now lives in a
  dedicated entity handler family.
- Loop, Workflow, Job, Evidence, Verification, Decision, Escalation, Profile,
  control, context, and planning orchestration is split into responsibility-
  specific handler modules.
- `src/pcl/cli.py` remains the global-option, routing, presentation, and error
  facade instead of owning every command body.

### Stable command and parser facades

- Domain mutations, next-action routing, and finish planning live in bounded
  service modules behind the existing `pcl.commands` import facade.
- Parser construction is split into ordered command-family builders behind one
  stable `build_parser()` entry point.
- Characterization tests preserve public imports, command/help structure,
  output, exit-code, typed-error, transaction, event, and zero-trace behavior.

## Compatibility and boundaries

- No command, flag, help, output, exit-code, SQLite, event/outbox, Evidence,
  human-gate, or generated-artifact contract changes intentionally.
- DB schema remains **8**; no migration is added.
- Runtime dependencies remain empty and Python support remains 3.10-3.13.
- Core remains local-only. No telemetry, hosted state, provider execution,
  automatic external mutation, or publication action is added.
