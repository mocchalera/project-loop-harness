# v0.3.3 draft release notes (guarded executor hardening)

## Guarded executor hardening

- `pcl workflow guard` is the primary dry-run and explicit execution surface.
- `pcl workflow sandbox` remains a warning-emitting compatibility alias through
  the `0.3.x` release line; migrate JSON consumers from `sandbox` /
  `workflow-sandbox/v1` to `guarded_executor` / `guarded-executor/v1`.
- Host execution now drains stdout/stderr incrementally, applies a default 1 MiB
  cap per stream, records original byte counts and truncation reasons, terminates
  process groups on timeout, and reports invalid UTF-8 as binary metadata.
- Evidence output uses shared conservative secret-shaped redaction before
  storage and records `redacted=true` when changed. This filter is not a secret
  scanner and does not prove that artifacts are secret-free.
- The permission contract now states argv-list execution, `shell=False`, the
  project-root working directory, environment allowlist inheritance, and the
  absence of OS, network, and filesystem isolation.
- Commands that require another parent variable can opt in by name with
  repeatable `--allow-env NAME`; recorded packets list names, never values.
