# project-loop-harness v0.3.3

**Release date:** 2026-07-10
**Scope:** v0.3.2 Master Trace / Intent Index + v0.3.3 Trust Foundation
(integrated roadmap Wave A). The v0.3.2 milestone ships inside this release;
there is no separate v0.3.2 tag.

Wave A originates from the integrated roadmap adopted 2026-07-10
(`docs/roadmap/integrated/ADOPTION.md`, decisions DEC-0001 / DEC-0002).

## Highlights

### Transactional audit outbox (0127 / 0128) — ADR-002, Accepted

- SQLite is now the single source of truth for domain state, events, and the
  outbox; `events.jsonl` is a projection delivered after commit. The previous
  dual-write (JSONL appended *before* the SQLite insert, commit owned by
  callers) could leak orphan JSONL lines on crash or rollback; it is gone.
- Migration **008** adds `events.sequence UNIQUE NOT NULL` and
  `outbox_records`. Schema version is now **8**.
- The projector appends canonical JSON lines in sequence order, marks rows
  delivered only after `fsync`, suppresses duplicates on retry, and never
  rewrites or truncates mismatching lines (`failed_needs_review` instead).
- `pcl audit flush` retries pending projections explicitly.

### Audit check / repair / rebuild (0129)

- `pcl audit check` (read-only), `pcl audit repair --dry-run|--apply`
  (preview-first), `pcl audit rebuild-jsonl --from-sqlite` (verified temp file,
  backup with SHA-256, atomic replace).
- Anomalies are classified `repairable` / `human_review` / `unsupported`; there
  is no silent repair, and unknown or legacy lines are isolated, never deleted.
- Exit semantics: 0 clean / 6 recoverable integrity issue / 7 unsupported /
  8 internal failure.
- Includes a fix for a false `ProjectionPendingError` immediately after
  concurrent commits.

### MCP specification compliance (0125 / 0126 + follow-up)

- stdio transport is now newline-delimited JSON-RPC per MCP 2025-06-18
  (previously Content-Length framing while declaring 2025-06-18). See
  `docs/releases/v0.3.3-mcp-stdio.md` for the breaking-change details; there is
  no legacy framing mode — roll back via the previous package version.
- Version negotiation no longer echoes unsupported requested versions.
- Initialization lifecycle is enforced: requests before `initialize` /
  `initialized` receive `-32002`; double `initialize` receives `-32600`;
  `ping` is allowed in all states.
- External conformance is proven against the official MCP Python SDK
  (`mcp==1.28.1`, optional `mcp-test` extra; tests skip with an explicit
  reason when the SDK is unavailable). Compatibility matrix:
  `docs/mcp-compatibility.md`; manual client runbook:
  `docs/mcp-manual-smoke-runbook.md`.

### Guarded executor hardening (0131)

- `pcl workflow guard` is the primary dry-run and explicit execution surface;
  `pcl workflow sandbox` remains a warning-emitting compatibility alias through
  the 0.3.x line. Migrate JSON consumers from `sandbox` / `workflow-sandbox/v1`
  to `guarded_executor` / `guarded-executor/v1`.
- Host execution drains stdout/stderr incrementally with a default 1 MiB cap
  per stream, records original byte counts and truncation reasons, terminates
  process groups on timeout, and reports invalid UTF-8 as binary metadata.
- Evidence output passes through shared conservative secret-shaped redaction
  (`src/pcl/redaction.py`, also used by the MCP server) and records
  `redacted=true` when changed. This filter is not a secret scanner.
- The permission contract states argv-list execution, `shell=False`,
  project-root cwd, environment allowlist inheritance (opt-in via repeatable
  `--allow-env NAME`; packets record names, never values), and the absence of
  OS/network/filesystem isolation.

### Master Trace / Intent Index (0123 / 0132 — v0.3.2 scope)

- `master-trace/v0`, `intent-index/v0`, and `master-trace-context/v0` are
  formalized as claims-not-facts evidence contracts
  (`docs/master-trace-intent-index.md`, accepted DEC-0002).
- `pcl context pack --task ... --master-trace-context` emits an opt-in additive
  section carrying evidence IDs and copied paths only — raw transcripts are
  never inlined. Absent or ambiguous evidence yields explicit
  `absent` / `selection_required`, never a fabricated choice.
- `pcl context check --json` always includes the factual preflight block for
  task targets (additive).

### Windows advisory-lock fallback (0133)

- The 0128 advisory locks initially required `fcntl`, which made every
  mutation (including `pcl init`) fail explicitly on Windows — caught by the
  Windows conformance CI job before tagging. Windows now uses an
  `msvcrt.locking` fallback with exclusive-only semantics: correctness and the
  ADR-002 exclusion contract are preserved, while normal mutations serialize
  on Windows (documented in `docs/event-outbox-compatibility.md`).
- Platforms with neither `fcntl` nor `msvcrt` still fail explicitly — locking
  is never silently skipped.
- CI now runs a Windows CLI smoke job (init / goal create / validate --strict /
  audit check) in addition to Windows MCP conformance.

### Reliability suite (0130) and baseline (0124)

- Deterministic crash-injection suite over the ADR-002 failure matrix
  (kill points at commit pre/post, JSONL append pre/mid/post, post-fsync,
  projector, migration statements, evidence temp/rename) plus 8-writer
  concurrency, ENOSPC/EACCES injection — ~10 s, part of the default suite.
  The fault hook requires double opt-in via `PCL_ENABLE_TEST_FAULTS=1` and
  `PCL_TEST_FAULT_POINT`; it cannot fire in normal execution.
- Found and fixed during 0130: a kill between evidence-manifest rename and DB
  commit previously left an unreferenced manifest that `audit check`
  misclassified as clean; it is now an `orphan_evidence_manifest` human-review
  anomaly.
- v0.3.1 CLI behavior is frozen as snapshot fixtures
  (`tests/fixtures/v0.3.1-baseline/`, environment-pinned generation); intended
  deltas since the freeze are recorded in that directory's README.

## Compatibility

- **DB schema 7 → 8** via migration 008 (forward-only; run `pcl migrate`).
  v0.3.3 refuses to run against newer schemas (existing downgrade guard).
- **MCP stdio framing is a breaking change** for any client that spoke the old
  Content-Length framing (none known besides the internal fixtures).
- Legacy `events.jsonl` files are preserved; new events append after the
  existing content. `pcl audit check` classifies pre-outbox history.
- `workflow sandbox` CLI remains as a deprecation alias for 0.3.x.
- Task IDs shipped: 0123–0133 plus the 0125 pre-initialize lifecycle follow-up
  (no spec file; tracked as T-0015 with evidence E-0040).
