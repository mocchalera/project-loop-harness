# v0.3.1 - Handoff Integrity + Operator Experience

Project Loop Harness v0.3.1 hardens the v0.3.0 target-bound context so a
"target-bound" label can never lie, and reduces the operator friction around
preflight, terminal close-out, human gates, and repeated coverage proposals.
No schema change; every addition is additive.

Handoff integrity:
- Target-bound receipt / link agreement. Context-pack selection and
  `validate --strict` now report `scope: target_bound` only when the selected
  receipt artifact's own `target_binding` agrees with the requested task or job.
  A routing row that disagrees with its artifact (hand-edited DB, migration
  accident) is caught: `pcl context pack --require-bound-receipt` fails with a
  new typed `context_pack_bound_receipt_mismatch` (distinct from
  `context_pack_bound_receipt_required`, so corruption is distinguishable from
  absence), the non-strict pack falls back to `unscoped_latest` instead of
  mislabeling, and strict validation flags the mismatch.
- Target-specific Markdown refresh. The code-context safety section's
  "Next action" now renders the same target-bound refresh command as the JSON
  `suggested_refresh_commands`, so an agent reading the Markdown pack is never
  told to run the unbound `pcl impact --diff --json`.

Operator experience:
- `pcl context check --task T-XXXX | --job J-XXXX` is a read-only handoff
  preflight. It reports whether a target-bound code-context receipt is
  `present` / `missing` / `mismatched` / `unavailable`, the supporting evidence
  count, and the canonical `pcl context pack ... --require-bound-receipt`
  command. With `--require-bound-receipt` it exits non-zero for scriptable
  gating. It writes nothing and states facts, not a go/no-go verdict.
- `pcl finish` plans the terminal close-out for the active run and its goal,
  reusing the existing `requires_human` / `safe_to_run` classification. It never
  asserts loop state (no verification, job/run completion, goal close); it hands
  those back as explicit commands. With `--execute` it runs only the safe
  generation tail (`validate --strict`, `render`), and only once no
  state-transition step remains.
- `pcl next` human-decision actions carry an additive `human_guidance_ja`
  ({`why_blocked`, `check`, `next_options`}) so the operator sees why the loop
  stopped, what to check, and the next options in Japanese. English fields are
  unchanged; agent-safe actions carry no Japanese block.
- `pcl loop run feature_coverage` no-ops when at least one feature exists and
  none are uncovered: it creates no run, jobs, or events and returns
  `no_op: true` / `reason: feature_coverage_already_covered`. Discovery (zero
  features) and any uncovered feature still start a real run; other workflows are
  unaffected.
- README and the adoption guide now surface the canonical target-bound handoff
  (`pcl impact --diff --for-task` then
  `pcl context pack --include-code-context --require-bound-receipt`).

Binding remains a caller assertion (`binding_strength: caller_asserted`), not a
semantic-relevance proof. No cognition or go/no-go vocabulary is added: the new
surfaces state what exists, not that any work is correct, sufficient, or safe.

Compatibility:
- No schema migration; schema version stays 7. No new runtime dependencies.
- Additive only: `pcl context check` and `pcl finish` are new commands; the
  `context_pack_bound_receipt_mismatch` error, `human_guidance_ja` field, and
  feature_coverage `no_op` result are additive. Existing JSON contracts and
  their fields are unchanged.
