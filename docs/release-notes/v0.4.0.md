# project-loop-harness v0.4.0

**Release candidate prepared:** 2026-07-10
**Status:** Local release candidate; not tagged, pushed, or published
**Scope:** Dogfood Operations + Three-command Wedge (Wave B)

This release turns the existing ontology-heavy workflow into a small operator
path: `pcl start` → `pcl finish --emit-packet` → `pcl resume`. It also adds the
measurement surface used to judge that path without adding telemetry or a
hosted service.

## Highlights

### Completion packet contract and guarded finish (0134 / 0135)

- `completion-packet/v1` now has a bundled JSON schema, validator, fixtures, and
  `pcl contract validate` surface.
- `pcl finish --emit-packet` snapshots the repository, runs only configured
  allowlisted checks, records bounded check Evidence, validates the packet, and
  commits the packet plus target transition transactionally.
- Existing plan-only `pcl finish` behavior remains backward compatible.

### Lite start and read-only resume (0136 / 0137)

- `pcl start "<intent>"` can initialize a project and create the first Goal,
  Task, start receipt, and audit event without requiring ontology knowledge.
- `pcl resume` emits deterministic `handoff-packet/v1` JSON or Markdown from
  current state and recorded Evidence. It does not mutate DB, events, outbox, or
  artifacts unless the caller explicitly requests an output file.

### Executable restart context repair (0139)

- `pcl evidence show E-XXXX [--json]` exposes bounded Evidence metadata without
  inlining bodies or executing claimed commands.
- Generated handoff packets add optional `restart_context`: target intent,
  explicit acceptance status, Evidence resolution commands, authoritative
  verification commands, changed paths, and documentation candidates.
- Terminal targets select the first authoritative reproducible passing check as
  the next safe replay command. When duplicate commands exist, the latest
  numeric completion check is authoritative.
- Old valid `handoff-packet/v1` documents without `restart_context` remain
  valid; the retained 0.3.3 minimal fixture proves this compatibility path.

### KPI report and dogfood operations (0138 / 0140)

- `pcl report kpi [--since YYYY-MM-DD] --json` aggregates local, read-only KPI
  data for verification feedback, context packs, finish packets, and handoff.
- Context-pack usage recording is explicit opt-in through `--record-usage`;
  default context generation remains read-only.
- Finish count and outcome distribution are derived from durable
  `completion_packet_created` events. Manual round-trip savings and unrecorded
  read-only resume operations remain `null` with factual reasons; no estimate is
  substituted.
- Two-repository measurements and boundaries are recorded in
  `docs/dogfood-report-v0.4.md`.

### RC2 lifecycle Integrity Gate (0140a / 0140b / 0140c)

- The three bundled `project-control-loop` Skill copies are byte-identical and
  their major direct-route commands are parser-tested against this CLI.
- Terminal Test and Feature transitions now accept canonical, hash-pinned
  `--evidence-id` proof. Missing, unhealthy, or mismatched Evidence is rejected
  before state, event, outbox, or JSONL mutation.
- A passing Test requires a same-Feature approved or waived Story. A done
  Feature requires reviewed Stories, complete non-waived Tests, no active
  Defects, and target-bound completion Evidence.
- A Goal can close through an approved same-Goal Workflow Verification or a
  valid same-target low-risk completed packet. The direct route also requires
  the Task created by `pcl start` to be terminal before Goal closure.
- Goal reports and dashboard data expose additive closure proof type, Evidence
  ID, Verification ID, and packet outcome fields.
- Obvious fail-open finish checks such as `|| true`, `|| echo`, `|| printf`,
  `|| :`, and `|| exit 0` are rejected with
  `fail_open_check_command` before execution.

## Compatibility and boundaries

- DB schema remains **8**; there is no migration in this release.
- Runtime dependencies remain empty; no hosted service, telemetry, LLM call, or
  automatic agent launch was added.
- `completion-packet/v1` and `handoff-packet/v1` are additive new contracts.
- `pcl resume`, `pcl evidence show`, and `pcl report kpi` are read-only by
  default. Replay commands are recorded claims/instructions, not newly executed
  facts.
- New projects default `validation.lifecycle_integrity` to `enforced`.
  Existing projects without the key receive advisory lifecycle findings for one
  release, while mutation guards still prevent new inconsistent terminal state.
- Legacy raw `--evidence` remains parseable with a warning, but it is not
  hash-pinned terminal proof and cannot replace an approved Verification or
  completed packet for Goal closure.
- Package/MCP version metadata changes from 0.3.3 to 0.4.0. The Codex plugin's
  independently versioned 0.1.12 manifest is unchanged.
- Task IDs shipped: 0134–0140c.
