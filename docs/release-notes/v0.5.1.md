# project-loop-harness v0.5.1

**Prepared:** 2026-07-15

**Status:** Published and independently verified

**Scope:** Trace & Efficient Handoff plus checkpoint-friction reduction

This release makes master-trace/intent-index handoff source-bound and keeps the
receiving agent's selected claims explicitly unverified until it checks the
copied source lines. It also makes checkpoint cadence advisory by default so a
time-based reminder does not become a human gate by itself.

## Highlights

### Deterministic Trace source binding

- `intent-index/v0` validation checks the copied Trace Evidence identity,
  member path, stored path, SHA-256, and one-based inclusive line ranges.
- Hash, Evidence, path, line-range, unsupported-contract, duplicate-item, and
  empty-source-ref failures produce stable diagnostics without mutating state.
- Valid context and resume packets expose bounded `trace_claim_refs` in
  deterministic order with omission metadata and `trust: unverified`.
- Invalid bindings omit claim refs and advertise
  `trace_claim_refs:invalid_binding`; full transcript bodies remain absent.
- Projects with no Trace/index retain the existing compatible resume shape.

### Controlled owner dogfood

- The accepted cohort `TRC-20260715-02` froze ten cases across
  `project-loop-harness` and `video-os-v2-spec` before execution.
- Independent Codex and Claude consumers resumed 6/6 valid bindings and safely
  stopped 4/4 broken bindings, with zero critical trust-boundary violations.
- No consumer received a full transcript or origin-session explanation.
- Representative packet/Trace byte ratios ranged from 0.344158 to 0.531553.
- This is controlled owner dogfood, not external-user adoption or market
  validation.

### Operator friction

- Checkpoint cadence is advisory by default, with explicit opt-in blocking and
  off modes.
- Existing completion, Evidence, dashboard, Council, and human-approval
  boundaries are unchanged.

## Compatibility and boundaries

- DB schema remains **8**; no migration is added.
- Runtime dependencies remain empty and Python support remains 3.10–3.13.
- Core remains local-only and does not call an LLM, capture transcripts,
  upload telemetry, or add hosted/cloud state.
- Council remains opt-in and provider execution remains outside Core.
- Historical v0.5.0 release notes, examples, and public URLs remain historical
  evidence and are not rewritten to v0.5.1.
