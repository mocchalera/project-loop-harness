# v0.4.3 cross-skill integrity dogfood

**Date:** 2026-07-11
**Project Loop:** G-0019 / F-0022 / T-0039 / TC-0050–TC-0055
**Status:** automated dogfood passed; independent human review pending

## Why this run exists

The LP-production flow in Cockpit task `cb004add` selected positive responsive
and page-flow reports while a strict coordinate report passed only 6 of 17
checks. The external skill called the result `prototype`, but PCL could still
reach `passing` and later `idle`. This clean-room run reproduces that evidence
shape without embedding web-specific semantics in PCL core.

## Canonical fixtures

- `tests/fixtures/cross_skill_dogfood/incomplete/` contains responsive success,
  a 6/17 coordinate failure, and a `prototype` completion verdict.
- `tests/fixtures/cross_skill_dogfood/complete/` contains the same declared
  report kinds with every report passing and a `complete` verdict.
- `tests/fixtures/cross_skill_dogfood/completion-policy.json` is a generic
  `completion-policy/v1` using only allowlisted JSON predicates.

## Observed results

### Incomplete prototype

- The failing coordinate report remained declared in the manifest but was not
  selected as positive Evidence.
- Because it was required, `evidence-set/v1` remained incomplete and emitted
  an exclusion warning.
- The completion-required Test rejected the transition with
  `completion_policy_failed`.
- Test status, Feature status, link count, event count, and outbox count were
  unchanged by the rejected transition.
- `pcl next --json` returned actionable unfinished work, never `idle`.

### Complete deliverable

- Responsive, coordinate, and completion-verdict reports were all included,
  required, passing, and hash-bound.
- The Evidence Set was complete and the completion policy evaluation passed.
- The Test and Feature reached `passing` through the Evidence-ID-first path.
- `pcl next --json` returned `review_passing_feature_completion`, preserving
  the distinction between `passing` and explicit `done`.
- A mediated approval receipt preserved human approver, agent recorder,
  conversation source reference, target, Evidence ID, and artifact SHA-256.

## Failures and limitations retained

- The first complete-fixture test used a nonconforming Work Brief ID and was
  correctly rejected as `work_brief_contract_invalid`; the fixture was fixed
  to a valid ID and rerun.
- PCL proves evaluation against declared, hash-bound reports. It does not prove
  that an external producer listed every report it should have produced.
- Conversation source references are caller-supplied provenance, not identity
  authentication or conversation-content capture.
- Motion, crop generation, typography inventory, and visual line detection
  remain external-skill concerns and were not implemented here.
- The automated fixture approval verifies contract mechanics. It is not the
  independent human release-gate approval for this dogfood report.
