# v0.3.3 MCP stdio compatibility note

`pcl-mcp --stdio` now uses the MCP 2025-06-18 standard transport: one compact
UTF-8 JSON-RPC message per newline-delimited line. This intentionally breaks
the non-standard `Content-Length` framing shipped in v0.3.1. No legacy framing
flag is provided; rollback is the previous package version.

The server supports protocol version `2025-06-18`. Initialize requests for
that version receive it unchanged. Requests for any unsupported version receive
the supported `2025-06-18` version instead of echoing the requested value.

Malformed, empty, or oversized input lines produce a compact JSON-RPC error
with a null response ID. The maximum input line size is 1 MiB. The server
discards the remainder of an oversized line before reading the next message,
so a bad message does not change subsequent framing. A partial message at EOF
produces a parse error; clean EOF ends the server.
Protocol responses are written only to stdout; diagnostics are written to
stderr.
