# v0.5.0 Council evaluation

The pre-result cohort, exclusions, and thresholds were frozen in commit
`ffb75c0` and Evidence `E-0194` (SHA-256
`fb30977621470f24990fadec9fe82ee0c2775060a989e1f96b6fc373a48264b9`).
No threshold was changed after execution.

## Result

- 12/12 cases exercised: 3 Direct route guards and 9 offline fixture/dry-run
  validations.
- Invalid/partial/budget/failed/skipped safe-stop: 6/6, 100%.
- Clear-task Council insertion: 0 accepted; Direct remains the default.
- `needs_human`: 3 projected Decisions, one per run and within the frozen cap.
- Invalid bundle persistence: 0.
- Local fixture + validation latency: p50 0.09 s, p95 0.12 s.
- Network/provider/paid use, tokens, and cost: 0.
- One stale request was rejected after the repository revision changed; a fresh
  deterministic prepare repaired it before cohort execution.

## Honest limitation

Quality, first-pass checks, rework, design drift, escaped defects, provider
latency, and human-attention savings are unavailable. Offline fixtures prove
the boundary and failure behavior, not the value of model deliberation. Model
ratings are therefore absent rather than invented.

## Recommendation

**Continue experiment.** Keep Council opt-in for ambiguous/high-risk work and
keep Direct as the clear-task default. A future adoption proposal needs at
least 10 frozen, paired, human-approved real-provider cases within the existing
privacy, budget, expiry, and revocation gates. Default changes, provider
selection, telemetry, publication, and distribution remain separate tasks.

The human owner must choose one outcome: adopt, adopt with constraints,
continue experiment, or reject. This report does not self-approve that gate.
