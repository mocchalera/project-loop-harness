# v0.5.1 controlled Trace resume runbook

Status: prepared; independent consumer execution is not yet authorized.

This runbook executes Task 0181 without external participants. Preparation is
local-only. It does not count as a resume result and does not establish model
quality, adoption, or cross-runtime success.

## Frozen inputs

- Cohort: `docs/evaluation/v0.5.1-trace-resume-cohort.json`
- Packets: `docs/evaluation/v0.5.1-trace-resume-packets/`
- Repository A: `project-loop-harness`
- Repository B: sibling `video-os-v2-spec`
- Threshold source:
  `tests/fixtures/trace_binding_v0/trace-resume-evaluation-fixture.json`

The cohort and every packet are SHA-256-bound before independent execution.
Changing a packet, case, threshold, repository assignment, or frozen check
invalidates the run and requires a new cohort ID.

## Authorization boundary

Independent model-backed sessions must not start until a human records:

1. session/runtime scope;
2. data class allowed to reach the model provider;
3. run budget;
4. authorization expiry.

The prepared cohort currently records all model/provider authorization flags
as false. Local packet generation and validation do not override those flags.

## Consumer isolation

Each consumer receives only:

- its assigned frozen packet files;
- packet-referenced copied Evidence artifacts;
- read-only repository state.

Do not provide this task transcript, prior assistant messages, a semantic
summary from the source session, or an unrecorded hint. Consumers must not edit
either repository. Result files are the only permitted write surface.

Assignments are frozen by `resume_session_id`:

- `resume-1`: TRE-001, TRE-003, TRE-005, TRE-007, TRE-009;
- `resume-2`: TRE-002, TRE-004, TRE-006, TRE-008, TRE-010.

The `runtime-a`, `runtime-b`, `model-a`, and `model-b` values are predeclared
cohort dimensions. They are not evidence that those boundaries were executed.
Each result must record the actual runtime/model or `not_executed`.

## Per-case procedure

1. Verify the packet SHA-256 against the cohort.
2. Validate the packet as `handoff-packet/v1`.
3. If `trace_claim_refs:invalid_binding` is omitted, stop before using any
   index claim. Record `outcome=safe_stop`, with no source line or repository
   verification command executed.
4. For a valid packet, keep every claim unverified, open only the copied source
   ranges named by `source_refs`, and compare the claim to those lines.
5. Run the repository slot's frozen verification command read-only.
6. Record the complete result contract. Do not infer success from packet shape
   alone.

Repository commands:

```text
A: PYTHONPATH=src pytest -q tests/test_trace_contract_fixtures.py
B: npm exec vitest run tests/validate.test.ts
```

## Result capture

Each case result must contain every field in
`required_result_fields` from the evaluation fixture, plus:

- `actual_resume_runtime`;
- `actual_resume_model`;
- `provider_authorization_ref`;
- `packet_sha256_checked`;
- `full_transcript_received`;
- `originating_session_explanation_received`.

`full_trace_size_bytes` means the copied Master Trace artifact size.
`handoff_size_bytes` means the packet's canonical `size_bytes` field; file
newline bytes are not added. A packet larger than its short controlled trace is
reported as observed and must not be hidden.

## Aggregate decision

After both result files validate, aggregate:

- valid-binding resume success denominator and rate;
- broken-binding required/observed safe stops and rate;
- critical trust-boundary violations;
- assistance count and all notes;
- packet/trace byte ratios for every case;
- no-index compatibility checks;
- actual independent-session and runtime/model coverage.

The report must recommend `continue`, `modify`, or `stop`. Task 0181 and the
v0.5.1 RC remain open until the human reviews that evidence and records one of
those decisions.
